# 1강: AI와 나는 어떤 관계인가?
## — 마인드셋 전환

> **소요시간**: 30분  
> **이 강의를 마치면**: Claude를 대하는 올바른 자세를 이해하고, 협업 모드가 무엇인지 설명할 수 있다

---

## Claude를 처음 쓸 때 흔히 하는 오해

### ❌ 오해 1: "시키면 다 해주는 비서"

혹시 이렇게 바이브 코딩 하고 계시진 않나요:

```
나: "앱을 만들꺼야. 소셜 로그인 PRD 작성해줘"
Claude: (일반적인 PRD 양식으로 작성)
나: "아 이게 아닌데... 내가 만들고 싶은 서비스는 이런 특성과 정책이 있고..."
Claude: (수정)
나: "아직도 아닌데..."
```

처음부터 다시 반복. 무슨 말을 하는지 모르겠어서, 결국 포기하고 내가 그냥 다 고치고 작성하게 됩니다.

### ❌ 오해 2: "잘 모르겠으면 더 자세히 설명하면 된다"

한 번에 모든 걸 다 설명하려고 엄청 긴 프롬프트를 씁니다. 그래도 결과가 기대와 다릅니다.

문제는 **길이가 아니라 순서**입니다.

### ✅ 올바른 이해: "같이 만들어가는 협업자"

Claude는 훌륭한 협업자입니다. 단, **내가 어떻게 협업하느냐에 따라 결과가 완전히 달라집니다.**

---

## AI와 협업하는 3가지 모드

### 모드 1: 단순 지시 (초보)

```
나: "마감 임박 알림 기능 PRD 써줘"
Claude: (일반적인 내용으로 작성)
결과: 우리 서비스와 맞지 않는 일반적인 PRD
```

**문제**: Claude가 우리 서비스를 모릅니다. 당연히 일반적인 답이 나옵니다.

---

### 모드 2: 조건부 지시 (중급)

```
나: "마감 임박 알림 기능 PRD 써줘.
     우리 앱은 풋살장 중개 플랫폼이고, 사용자는 2~30대 남성이 많아.
     친구, AI 리포트 배치 케이스를 포함해줘."
Claude: (조건을 반영해서 작성)
결과: 조금 더 맞는 PRD, 하지만 여전히 수정이 많이 필요
```

**문제**: 조건을 줬지만, 컨텍스트가 부족합니다. 수정이 계속 필요해요.

---

### 모드 3: 컨텍스트 기반 협업 (고수)

```
나: "우리 서비스 설명할게. 이해한 내용 요약해줘."
Claude: "이해했습니다. 확인할 게 몇 가지 있는데..."
나: (대화로 컨텍스트 축적)

나: "더 많은 사용자들이 빠르게 마감되는 매치를 탐색하고 신청하여 풀 부킹할 수 있어야 해.
     어떤 접근 방법들이 있을지 먼저 탐색해줘."
Claude: "다음과 같은 접근법들이 있습니다..."
나: (방향 확정)

나: "이제 이 방향으로 PRD 초안 작성해줘."
Claude: (맥락을 충분히 이해한 상태로 작성)

나: "작성한 PRD를 개발자 관점에서 비판해줘."
Claude: (스스로 검토 후 개선점 제안)
결과: 팀에 바로 공유할 수 있는 PRD
```

**차이**: 처음부터 결과물을 요구하지 않고, **먼저 Claude가 우리 상황을 이해하게 만들었습니다.**

---

## 왜 이 차이가 중요한가?

Claude를 쓰면서 가장 중요하게 깨달은 것:

> **내가 직접 결과물을 만드는 게 아니라,
>  결과물을 잘 만드는 시스템을 만드는 것.**

PO 입장으로 번역하면:

> **내가 PRD를 쓰는 게 아니라,  
>  PRD를 잘 써주는 시스템을 만드는 것**

한 번 시스템을 잘 만들어두면, 다음 PRD는 훨씬 빠르고 정확하게 나옵니다.  
이렇게 만들어진 루틴을 활용하여 더욱 가속화 할 수 있게 됩니다.

---

## Claude를 대하는 올바른 태도

### 1. 결과물이 아닌 과정을 설계하라

❌ "PRD 완성본 줘"  
✅ "PRD 작성 과정을 함께 밟아나가자"

### 2. 한 번에 완성하려 하지 마라

❌ 긴 프롬프트 하나로 완벽한 결과 기대  
✅ 탐색 → 구조 → 초안 → 검토 → 완성, 단계별로

### 3. Claude의 답에서 배워라

Claude가 "이런 점도 고려해야 합니다"라고 말할 때, 그게 실제로 내가 놓친 부분일 수 있습니다. 무시하지 말고 활용하세요.

### 4. 틀려도 괜찮다

Claude의 첫 번째 결과물이 완벽하지 않아도 됩니다. 피드백을 주면서 함께 개선하는 게 올바른 방식입니다.

---

## 자주 묻는 질문

**Q: 한국어로 써도 되나요?**  
A: 네. 작년 11월 즈음 부터는 품질 차이가 크게 나지 않습니다. 오히려 한국어로 쓰는 게 더 빠르고 자연스럽습니다.

**Q: 존댓말을 써야 하나요?**  
A: 꼭 그럴 필요는 없습니다. 다만 "AI에게 반말을 하다 보면 사람에게도 반말 습관이 생긴다"는 말이 있어요. 저는 그래서 존댓말을 씁니다. 본인이 편한 방식으로 하세요.

**Q: Claude가 틀린 내용을 말하면 어떻게 하나요?**  
A: 당연히 고쳐줘야 합니다. "방금 말한 내용 중 [이 부분]은 우리 상황과 다릅니다. [올바른 내용]으로 수정해서 반영해주세요"라고 하면 됩니다. AI 도구를 쓰는 것에 있어서 너무너무 중요한 액션입니다.

**Q: 요금이 걱정됩니다.**  
A: Claude.ai 무료 플랜도 PRD 작성 용도로는 충분합니다. 많이 쓰게 되면 Pro 플랜($20/월) 고려해보세요.

## ✏️ 오늘의 실습 과제

**목표**: Claude와 첫 대화를 나눠보기

**따라하기**:

1. Claude.ai 접속 (claude.ai)
2. 새 대화 시작
3. 아래 메시지를 입력해보세요:

```
안녕하세요. 저는 [회사/서비스명]의 PO입니다.
오늘 처음으로 Claude를 업무에 활용해보려고 합니다.

현재 가장 어려운 업무가 [PRD 작성 / 기능 기획 / 회의 준비 등]인데요.
Claude가 어떻게 도움을 줄 수 있을지 간단히 설명해주시고,
시작하기 전에 저에게 어떤 정보가 있으면 좋을지 알려주세요.
```

4. 답변을 읽고 느낀 점 메모하기:
   - 예상보다 잘 됐던 것
   - 어색했던 것
   - 다음에 시도해보고 싶은 것
