# 3강: Soul Document 만들기
## — 나만의 CLAUDE.md 작성

> **소요시간**: 45분  
> **이 강의를 마치면**: 자신의 프로젝트에 맞는 Soul Document 초안을 Claude와 함께 만들 수 있다

---

## Soul Document가 왜 필요한가?

### Claude는 매 대화가 새로 시작됩니다

Claude는 이전 대화를 기억하지 못합니다. 어제 "우리 서비스는 30대 여성 대상 패션 커머스야"라고 했어도, 오늘 새 대화를 시작하면 다시 처음부터 설명해야 합니다.

만약 이 설명을 매번 한다면:
- 매일 5~10분씩 동일한 설명 반복
- 설명을 빠뜨리면 엉뚱한 결과가 나옴
- PRD 스타일이 매번 달라짐

### Soul Document로 이 문제를 해결합니다

CLAUDE.md는 일종의 soul document입니다.
Claude Code를 쓰든 Claude.ai를 쓰든,
이 폴더에서 작업을 시작하게 되면 항상 맨 처음 읽히는 파일입니다.

**Soul Document = 대화를 시작하기 전에 Claude에게 먼저 읽게 하는 나만의 안내서**

한 번 잘 만들어두면:
- 매번 긴 설명 없이도 맥락을 이해한 답변을 받음
- 우리 서비스에 맞는 스타일로 일관되게 작성
- 새 프로젝트 시작할 때 업데이트만 하면 됨

---

## Soul Document가 있을 때와 없을 때의 차이

### Soul Document 없이:

```
나: "충전 실패 알림 PRD 써주세요"

Claude: (일반적인 결제 시스템 기준으로 작성)
        → 우리 서비스 특성과 다른 내용
        → 우리 팀이 쓰지 않는 용어 사용
        → 우리 사용자와 다른 페르소나 가정
```

수정에 시간이 오래 걸림.

---

### Soul Document가 있을 때:

```
[대화 시작 전 또는 Project Instructions에 Soul Document 입력]

나: "충전 실패 알림 PRD 써주세요"

Claude: (Soul Document를 읽고)
        → 우리가 커머스 앱이고 30대 여성 사용자임을 알고 작성
        → 우리 팀이 쓰는 용어와 형식으로 작성
        → 기존 PRD 스타일과 일관된 형식으로 작성
```

수정이 최소화됨.

---

## PO를 위한 Soul Document 구조

아래는 PO가 만들어야 할 Soul Document의 기본 구조입니다. 템플릿 파일은 `templates/CLAUDE_md_template.md`에 있습니다.

```markdown
# [서비스명] PO 작업 가이드

## 나(PO)에 대해
- 역할: [PO / 시니어 PO / 프로덕트 매니저]
- 도메인 경력: [커머스 3년, 핀테크 2년 등]
- 주로 작성하는 문서: [PRD, 기획서, 스프린트 리뷰 등]

## 우리 서비스에 대해
- 서비스명: [이름]
- 한 줄 설명: [무엇을 하는 서비스인가]
- 현재 단계: [베타 / 정식 출시 / 스케일업 중]
- 주요 사용자: [누구인가, 나이/성별/특성]
- 핵심 사용자 문제: [무엇을 해결해주는가]
- 경쟁 서비스 대비 차별점: [왜 우리를 쓰는가]

## 우리 팀에 대해
- 팀 규모: [개발 2명, 디자인 1명, PO 1명 등]
- 개발 방식: [2주 스프린트 / 칸반 등]
- 주요 기술 스택: [React, Node.js, Python 등 - 대략적으로만]

## 문서 작성 원칙
- 항상 사용자 관점에서 먼저 서술한다
- 기술 용어는 비개발자도 이해할 수 있게 풀어서 쓴다
- 엣지케이스는 별도 섹션으로 항상 포함한다
- 불명확한 사항은 Open Questions로 표시한다
- 한국어로 작성한다

## 자주 쓰는 용어 정의
- [우리 서비스 내부 용어]: [설명]
- [다른 서비스와 다른 의미의 용어]: [우리 정의]

## 현재 진행 중인 컨텍스트
- [현재 스프린트 목표 또는 분기 목표]
- [최근 결정된 주요 방향]
```

---

## Soul Document 작성 방법

### 방법 1: Claude와 대화하며 만들기 (추천)

직접 작성하기 어려우면, **Claude에게 대화로 만들어 달라고 하면 됩니다.**

아래 프롬프트를 복사해서 쓰세요:

```
나는 [회사명]의 PO입니다.

우리 서비스: [서비스 설명]
주요 사용자: [사용자 설명]
내가 주로 작성하는 문서: PRD, 기획서, 스프린트 리뷰, 회의록 등

이 정보를 바탕으로, 앞으로 나와 Claude가 잘 협업하기 위한 
CLAUDE.md (Soul Document) 초안을 작성해주세요.

다음 섹션들을 포함해주세요:
- 나와 서비스에 대한 요약
- 문서 작성 원칙
- 자주 쓸 용어 정의 공간

작성하기 전에, 더 알아야 할 정보가 있으면 먼저 질문해주세요.
```

### 방법 2: 템플릿으로 직접 작성하기

`templates/CLAUDE_md_template.md` 파일을 열어서 `[대괄호]` 부분을 채우세요.

---

## Soul Document 활용하는 법

### Claude.ai Projects 사용하는 경우 (권장)

1. Claude.ai 접속
2. 왼쪽 메뉴 → **Projects** 클릭
3. **New Project** 생성 (서비스명으로)
4. **Project Instructions** 탭 → Soul Document 내용 붙여넣기
5. 이후 이 Project 안에서 대화하면 항상 Soul Document를 읽은 상태로 시작

### 일반 대화에서 사용하는 경우

대화 첫 번째 메시지에 아래를 붙여넣으세요:

```
아래는 내 작업 가이드입니다. 읽고 이에 맞게 도와주세요.

---
[Soul Document 전체 내용 붙여넣기]
---

준비됐으면 "준비됐습니다"라고 말해주세요.
```

## Soul Document를 잘 만드는 팁

### 팁 1: 처음부터 완벽하게 만들려 하지 마라

처음에는 10줄짜리 간단한 버전으로 시작하세요. 쓰다 보면 뭘 추가해야 할지 자연스럽게 알게 됩니다.

### 팁 2: 쓸수록 업데이트하라

새로운 결정이 생기거나, 자주 반복하는 설명이 생기면 바로 Soul Document에 추가하세요.

```
[업데이트 예시]
- 오늘 팀에서 결정: 알림 기능은 앱 전용으로만 (웹 제외)
→ Soul Document에 추가: "알림 관련 기능은 모바일 앱에만 적용됩니다"
```

### 팁 3: 용어 정의를 꼭 넣어라

내부에서 쓰는 특별한 용어들을 정의해두면, Claude가 정확한 맥락으로 이해합니다.

```markdown
## 용어 정의
- "파트너": 우리 플랫폼에 입점한 셀러 (일반 사용자와 구분)
- "에디터픽": 운영팀이 직접 선별한 큐레이션 상품
- "스텝": 온보딩 프로세스의 각 단계
```

### 팁 4: "하지 말 것"도 적어라

```markdown
## 작업 시 주의사항
- 기술 구현 세부사항은 PRD에 포함하지 않는다 (개발팀이 결정)
- 우선순위는 항상 Must Have / Nice to Have 로 구분한다
- 출시일은 PRD에 명시하지 않는다 (별도 일정표에서 관리)
```

## 실제 Soul Document 예시

아래는 완성된 Soul Document 예시입니다. 참고해서 자신의 것을 만드세요.

```markdown
# StylePick PO 작업 가이드
## CLAUDE.md — 최종 업데이트: 2026년 2월

---

## 나에 대해
- 역할: PO (Product Owner)
- 경력: 패션 커머스 3년, 이전 일반 커머스 2년
- 주요 업무: PRD 작성, 스프린트 기획, 이해관계자 커뮤니케이션

## 우리 서비스
- 서비스명: StylePick
- 설명: 30대 여성을 위한 AI 기반 패션 코디 추천 커머스
- 단계: 정식 출시 후 스케일업 중 (MAU 50만)
- 주요 사용자: 30-35세 직장 여성, 트렌드 관심 높음, 쇼핑에 시간 투자 어려움
- 핵심 문제 해결: "뭐 입을지 모르겠어요" 해결, 코디 제안으로 구매 결정 속도 up
- 차별점: 체형/피부톤 기반 개인화 추천 (경쟁사 대비 반품률 30% 낮음)

## 팀 구성
- 개발: iOS 2명, Android 1명, 백엔드 2명, 프론트 1명
- 디자인: 1명 (UX/UI 모두)
- 개발 방식: 2주 스프린트, Jira 사용, Slack 소통

## 문서 작성 원칙
- 모든 기능은 "30대 직장 여성이 점심시간 5분 안에 사용한다"는 기준으로 설계
- 기술 용어 대신 사용자 행동으로 서술 ("API 호출"이 아닌 "화면이 자동으로 갱신됨")
- 엣지케이스 섹션 필수 포함
- 미결 사항은 Open Questions 섹션에 명시
- 한국어 작성

## 주요 용어 정의
- "스타일": 코디 아이템 조합 (단품 상품과 구분)
- "큐레이션": 에디터가 직접 선별한 스타일/상품
- "매칭 점수": AI가 계산한 사용자-스타일 적합도 (0-100점)
- "저장": 위시리스트에 추가하는 행동
- "팔로우": 특정 에디터/브랜드 구독

## 현재 컨텍스트 (2026년 2월)
- Q1 목표: 재구매율 15% 향상
- 현재 스프린트: 장바구니 개선 (3/15 출시 목표)
- 최근 결정: 남성 카테고리는 Q2로 연기
```

## ✏️ 오늘의 실습 과제

**목표**: 나만의 Soul Document 초안 완성하기

**Step 1**: 아래 프롬프트를 Claude에게 보내세요

```
안녕하세요. 저는 [회사/서비스명]의 PO입니다.

저와 앞으로 잘 협업하기 위한 Soul Document(CLAUDE.md)를 
만들고 싶어요. 

먼저 저에 대해 파악하기 위해 필요한 질문들을 해주세요.
최대 5가지 질문으로 핵심만 물어봐줘요.
```

**Step 2**: Claude의 질문에 답하면서 정보 제공

**Step 3**: 정보 제공 후 아래 요청

```
지금까지 나눈 대화를 바탕으로 CLAUDE.md 초안을 작성해주세요.
위에서 말한 섹션 구조를 포함해주세요.
```

**Step 4**: 초안을 검토하고 수정하기  
빠진 내용, 틀린 내용 수정 후 파일로 저장

**완성된 파일 저장 위치**: 자신이 자주 작업하는 폴더 또는 노션/구글 독스에 보관
