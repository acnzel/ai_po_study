## 진행 상황

> `[ ]` = 미완료, `[x]` = 완료, `[-]` = 진행 중

### PRD 작성 플로우

- [ ] **Phase 1: 탐색** — 기능 배경, 유사 사례, 고려 요소 조사
- [ ] **Phase 2: 구조** — PRD 섹션 구조 확정
- [ ] **Phase 3: 초안** — 각 섹션 내용 작성
- [ ] **Phase 4: 검토** — 개발자 / 사용자 / 비즈니스 관점 검토
- [ ] **Phase 5: 완성** — 피드백 반영 최종본
- [ ] **공유** — 팀에 공유 완료
