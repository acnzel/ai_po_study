# 8강: 내 Workflow 완성하기
## — 종합 실습

> **소요시간**: 60분  
> **이 강의를 마치면**: 1~7강의 모든 내용을 통합한 나만의 PO-Claude Workflow를 갖추고, 실제 PRD 1개를 완성할 수 있다

---

## 여기까지 온 당신에게

1강부터 7강까지 배운 것들을 돌아봅시다.

| 강의 | 배운 것 | 실무에서 쓰는 장면 |
|------|--------|-----------------|
| 1강 | AI와의 올바른 관계 | 매번 대화 시작 전 자세 잡기 |
| 2강 | 프롬프팅 5패턴 | 어떤 작업이든 이 패턴으로 시작 |
| 3강 | Soul Document | 프로젝트 폴더마다 CLAUDE.md 유지 |
| 4강 | PRD 5단계 플로우 | 새 기능 기획할 때마다 |
| 5강 | 컨텍스트 쌓기 | 복잡한 작업 시작 전 20분 투자 |
| 6강 | Command 자동화 | 반복 업무가 생길 때마다 Command 추가 |
| 7강 | Agentic Workflow | 중요한 기획은 에이전트 4단계로 |

이제 이 모든 것을 **하나의 일관된 Workflow**로 묶을 차례입니다.

---

## 나만의 PO-Claude Workflow 설계

### Workflow 전체 지도

```
📌 새 프로젝트/기능 기획 시작
│
├─ [준비] Soul Document 확인 & 업데이트
│         PROGRESS.md 생성
│
├─ [탐색] 에이전트 1: UX 리서처 역할로 배경 조사
│         ↓ 20분 컨텍스트 쌓기 대화
│
├─ [기획] 에이전트 2: PRD 초안 작성
│         (4강 5단계 플로우 적용)
│
├─ [검토] 에이전트 3: 다각도 비판 검토
│         개발자 / 사용자 / 비즈니스 관점
│
├─ [완성] 에이전트 4: 피드백 반영 최종본
│
└─ [공유] 팀 공유 전 내 최종 검토
          → Slack / Notion / Jira에 업로드

📌 반복 업무 (Command 사용)
│
├─ /sprint-review    → 매 스프린트 종료 시
├─ /feature-analysis → 기능 요청 들어올 때
├─ /prd-draft        → 빠른 초안 필요할 때
└─ 기타 나만의 Command들

📌 주기적 관리
│
├─ Soul Document 업데이트 (월 1회)
└─ Command 개선 (분기 1회)
```

## 실전 워크플로우 시나리오

### 상황: 월요일 아침, 새 기능 기획 요청이 들어왔다

> "이번 스프린트에 '찜하기 기능 개선'을 추가해줘.  
>  사용자들이 찜한 상품이 품절됐을 때 재입고 알림을 받고 싶어한다는 피드백이 많아."

---

#### 월요일 오전 (30분): 준비 & 탐색

**Step 1: Soul Document 확인**
```
현재 CLAUDE.md를 열어서 확인:
- 현재 스프린트 목표가 맞게 기재되어 있는지
- 이번 분기 KPI가 업데이트되어 있는지
- 필요하면 수정 후 저장
```

**Step 2: PROGRESS.md 생성**
```
# 찜하기 재입고 알림 기능 PRD

## 목표
Q1 전환율 목표 달성을 위한 재방문 유도 기능

## 요청 배경
사용자 피드백 다수 → 찜한 상품 품절 후 재입고 알림 필요

## 진행 상황
- [ ] 탐색
- [ ] PRD 초안
- [ ] 검토
- [ ] 최종 완성
```

**Step 3: 탐색 에이전트 실행**
```
찜하기 기능에서 재입고 알림 기능을 추가하려 해.
UX 리서처 관점에서 탐색해줘:

1. 커머스에서 재입고 알림의 일반적인 구현 방식
2. 사용자가 기대하는 알림 경험 (언제, 어떻게, 얼마나 자주)
3. 이 기능이 전환율에 미치는 영향 (알려진 데이터가 있으면)
4. 기획 시 놓치기 쉬운 케이스들

[CLAUDE.md 내용 붙여넣기]
```

---

#### 월요일 오전 (20분): PRD 초안

**Step 4: 초안 에이전트 실행**
```
탐색 내용을 바탕으로 PRD 초안을 작성해줘.

추가 컨텍스트:
- 이번 스프린트 여유 개발 시간: 개발자 1명 x 1주
- 우선 필요한 건: 푸시 알림 1개 (이메일은 다음으로)
- 찜한 상품이 100개 이상인 사용자 많음 (알림 피로도 주의)
```

---

#### 월요일 오후 (20분): 검토 & 완성

**Step 5: 검토 에이전트 실행**
```
이 PRD를 두 관점에서 검토해줘:

[iOS 개발자]:
"이대로 개발 시작하면 뭘 더 물어봐야 하나?"

[찜을 100개 이상 한 헤비 유저]:
"이 알림을 받으면 좋겠는가, 귀찮은가?"
```

**Step 6: 최종 완성**
```
검토 결과 반영해서 최종 버전 만들어줘.
특히 알림 피로도 관련 설정 옵션 섹션을 보강해줘.
```

**Step 7: 내 최종 검토 후 팀 공유**

---

**총 소요 시간**: 약 70분  
**결과물**: 팀에 바로 공유 가능한 PRD 1개

---

## 최종 체크리스트: 초보 PO를 위한 Claude 활용 10계명

커리큘럼을 마치며, 앞으로 기억해야 할 것들입니다.

```
✅ 1. Soul Document를 만들고 꾸준히 업데이트한다
      → 한 번 잘 만들어두면 매번 설명이 필요 없다

✅ 2. 한 번에 다 요구하지 않는다 — 탐색 먼저
      → 첫 요청은 항상 "탐색"이나 "구조 제안"으로 시작

✅ 3. 결과물 생성 전 컨텍스트를 충분히 쌓는다
      → 20분 대화가 2시간 수정을 막는다

✅ 4. 초안이 나오면 바로 다각도 검토를 요청한다
      → 개발자 / 사용자 / 비즈니스 세 관점은 필수

✅ 5. 에이전트에게 자기 결과물을 비판하게 만든다
      → "이 PRD의 가장 큰 문제 3가지 찾아줘"

✅ 6. 반복 업무는 Command로 만들어 자동화한다
      → 두 번 이상 같은 설명을 했다면 Command 만들 때

✅ 7. 긴 작업은 PROGRESS.md로 중간 저장한다
      → 내일 새 대화에서도 이어받을 수 있다

✅ 8. 최종 판단은 항상 내가 한다
      → Claude는 초안과 검토를 돕는 도구, 결정은 PO가

✅ 9. 효과적인 프롬프트는 반드시 기록해둔다
      → Command 파일에 저장, 다음에도 재사용

✅ 10. 안 되는 게 있으면 2주 후에 다시 시도한다
       → Claude가 빠르게 발전 중, 안 됐던 것도 곧 된다
```

## ✏️ Final Project: 실제 PRD 1개 완성하기

**목표**: 이 커리큘럼에서 배운 모든 것을 적용해서 실제 PRD 1개 완성

### 조건

```
□ 자신의 현재 프로젝트에서 가장 중요한 기능 선택
□ Soul Document를 대화 첫머리에 활용
□ 탐색 → 초안 → 검토 → 개선 4단계 에이전트 플로우
□ 개발자 / 사용자 / 비즈니스 3관점 검토 완료
□ 엣지케이스 섹션 포함 (최소 5개)
□ Open Questions 섹션에 미결 사항 기록
□ 총 소요 시간 90분 이내 목표
```

### 완성 후 제출물

```
1. 완성된 PRD 파일
2. 사용한 주요 프롬프트 3개 (가장 효과적이었던 것)
3. 학습 회고:
   - 잘 됐던 것
   - 어려웠던 것
   - 다음에 개선할 것
   - Claude 없이 했을 때와 비교해서 어떻게 달랐나?
```
